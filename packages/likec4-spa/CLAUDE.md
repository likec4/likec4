# Agent Instructions

## Subdirectory layering

This package is the webapp host (`@likec4/spa`) — it materializes `@likec4/diagram` into a runnable Vite SPA. Imports flow upward only.

- `src/routes/` — TanStack Router route definitions. `routeTree.gen.ts` is generated by `tsr generate` (run via `pnpm generate`); do not edit it.
- `src/pages/` — page-level components, one per route. Owns async data wiring. Import from `src/context/` and `likec4:*` virtual modules. Do not import sibling pages.
- `src/components/` — reusable UI. Safe to import from pages; do not import pages from here.
- `src/context/` — React providers that bridge `likec4:*` virtual modules into `@likec4/diagram`. `LikeC4ModelContext.tsx` is where the diagram's `LikeC4ModelProvider` contract is fulfilled (atom + provider). New context that adapts virtual-module data goes here.
- `src/main.tsx`, `src/router.tsx` — entrypoints; `start-dev.ts` is the dev-server bootstrap invoked by `pnpm dev`.
- `lib/`, `dist/`, `codegen/` — build output / generators consumed by `packages/likec4` for the static-site build. Do not edit by hand.

## State management

- **`nanostores`** — the package's primary reactive primitive. Use atoms for state that crosses route boundaries or needs to survive HMR (e.g. the model atom in `context/LikeC4ModelContext.tsx`, the sidebar drawer in `components/sidebar/state.ts`). Keep atom modules small and HMR-friendly.
- **React `useState` / `useReducer`** — local UI state inside a single page or component (form inputs, hover, expansion).
- **No XState in this package.** Diagram-side state machines live in `@likec4/diagram`; do not add machines here.

## Public API & contracts

- `package.json` is `private: true`. This is an app, not a library — there are no exports to keep stable.
- Talk to the Language Server only through `likec4:rpc` from `@likec4/vite-plugin`. Do not import `@likec4/language-server` or `@likec4/vite-plugin` source directly.
- All cross-process data comes from `likec4:*` virtual modules (`likec4:model`, `likec4:projects`, `likec4:app-config`, `likec4:icons`, `likec4:rpc`, format exporters). Adding a new data source means adding a virtual module in `@likec4/vite-plugin`, not fetching here.
- New routes require regenerating `routeTree.gen.ts` (`pnpm generate`).
